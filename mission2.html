Mission 2 – Interface low code

Exploitation de la base KDou avec Microsoft Access

Objectifs de la mission

Dans cette mission, nous avons utilisé Microsoft Access comme outil low code pour interroger la base KDou sans écrire directement du SQL.

  • Créer des requêtes simples et multi-tables avec QBE
  • Appliquer des tris, filtres et critères
  • Visualiser les jointures entre les tables
  • Analyser les résultats pour aider à la décision

Requêtes réalisées

Produits par fournisseur

But : savoir quels produits sont fournis par quels fournisseurs.

Tables : Fournisseur, Produit

Résultat : liste des produits avec le nom du fournisseur et la catégorie.

Commandes par client

But : analyser le comportement d'achat de chaque client.

Tables : Clientèle, Commande

Résultat : société, ville, n° commande, date de commande.

Produits jamais commandés

But : repérer les produits du catalogue qui ne génèrent pas de ventes.

Tables : Produit, DétailCommande

Résultat : référence produit, nom, fournisseur, catégorie.

Clients pour livraisons groupées

But : identifier des groupes de clients situés dans la même ville.

Tables : Clientèle, Commande

Résultat : paires de clients avec commandes proches dans le temps.

Analyse de l'outil Access

Points forts

  • Interface graphique intuitive
  • Création de requêtes sans écrire de SQL
  • Visualisation des relations entre tables
  • Génération automatique du code SQL

Limites

  • Dépendance au système Windows
  • Peu adapté aux très grosses bases de données
  • Requêtes avancées parfois difficiles à construire en mode graphique
  • SQL généré spécifique à Access

Apport pour KDou

Grâce à cette mission, KDou peut :

  • Analyser rapidement ses données sans experts SQL
  • Explorer son catalogue et ses clients de manière interactive
  • Préparer la transition vers des outils plus avancés (SQL pur, BI...)
